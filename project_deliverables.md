Required Deliverables
Submit one GitHub repository that contains your complete Project 3 work. The repository should make it clear how your autonomous research agent works, how to run it, and how it meets the project criteria.

1. GitHub Repository
Your repository must include:

A clear README.md with the project overview, setup instructions, required environment variables, and run steps
Organized source files for your agent, workflow, scripts, or n8n export
A dependency file such as requirements.txt, pyproject.toml, or clear n8n setup notes
Example .env.example file that lists required keys without exposing secrets
Meaningful commit history that shows project progress
A skills/ directory containing all skills used or created for the project
2. Working Autonomous Agent
Your implementation may use Python, LangChain, LangGraph, n8n, MCP tools, direct API calls, or a justified hybrid architecture. It must include:

An agent-based workflow that runs from trigger/input to final report output
At least 2 real tool/API integrations
A clear explanation of why you chose your framework or workflow approach
Error handling, retry logic, and validation for tool/API interactions
Optional advanced components if useful for your use case, such as RAG, MCP, LangGraph state routing, or n8n orchestration
3. Agile Planning Artifacts
Organize the project using Agile user stories and a sprint plan. At minimum, include docs/stories.md in your repository. GitHub Issues, Asana, or Jira are also valid if you link them clearly from the README.md.

Your planning artifacts must include:

User stories that describe the project from the user's perspective
Sprint tasks connected to those stories
An estimate for how hard each task is
Dependencies between tasks
A definition of done for each task you ran with the agent
4. Agent Specification and Instructions
Include the instructions and context you used to guide agent work:

The project specification as written for the agent
Every AGENTS.md file you created
Any reusable project skills under skills/
Notes on which tasks were delegated to the agent and how you reviewed the result
5. Generated Report Examples
Include 2-3 sample reports generated by your agent. These reports should:

Use different companies, brands, topics, or research questions
Show that the agent can synthesize information from multiple sources
Follow a structure that fits your selected use case
Make clear where the information came from when possible
6. Documentation
Add documentation that helps an instructor or reviewer understand your design:

Architecture overview or diagram
Workflow explanation from trigger/input to final report
Tool/API descriptions and authentication requirements
Known limitations, failure modes, or cost/rate-limit considerations
Testing or validation notes
7. Demo Video or Live Presentation
Prepare a 5-7 minute demo that shows:

The agent receiving an input or trigger
Tool/API calls or workflow steps in action
The generated report output
Key architecture decisions and tradeoffs
One reliability feature, such as validation, retry logic, or error handling
How your Agile plan, task definitions of done, agent specification, and skills/ directory are organized
Submission
Submit your GitHub repository link as specified by your instructor. Do not submit secrets, API keys, standalone Google Docs, or cloud-only links as the primary artifact.
