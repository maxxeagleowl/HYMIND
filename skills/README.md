# Skills

This folder contains reusable guidance files that shape how Codex works on HYMIND.
Skills are split into governance and operational categories so system rules stay separate from domain execution rules.

## Governance Skills

- `governance/skill_agent_memory.md` - preserve project memory and continuity across sessions.
- `governance/skill_agile_execution.md` - keep planning, execution, and validation synchronized.
- `governance/skill_api_integration.md` - define standards for reliable external API use.
- `governance/skill_langgraph_workflow.md` - define modular LangGraph workflow standards.
- `governance/skill_project_documentation.md` - keep repository documentation clear and current.
- `governance/skill_rag_pipeline.md` - define retrieval and historical intelligence standards.
- `governance/skill_reliability_engineering.md` - enforce stability, observability, and recoverability.

## Operational Skills

- `operational/skill_hydrogen_research.md` - gather and structure hydrogen industry intelligence.
- `operational/skill_report_generation.md` - define executive report generation standards.
- `operational/skill_report_generation_ops.md` - turn validated findings into concise reports.
- `operational/skill_source_validation.md` - check freshness, credibility, and duplication.
- `operational/skill_market_monitoring.md` - track market signals and watchlists.
- `operational/skill_executive_reporting.md` - shape outputs for executive consumption.

## Usage

- Use the most specific skill that matches the task.
- Keep each skill focused on one responsibility.
- Prefer short, concrete instructions over broad prose.
- Update a skill when the workflow changes instead of adding duplicate guidance elsewhere.

## Project Notes

- Skills in these subfolders are treated as reusable operational guidance for the project.
- Keep them practical, current, and aligned with the active project phase.
- Preserve clarity over completeness; a skill should be easy to apply at a glance.
