# Skills

This folder contains reusable guidance files that shape how Codex works on HYMIND.
Each skill is a short, task-specific instruction set that can be reused across phases
without repeating the same guidance in every task.

## Current Skills

- `skill_agent_memory.md` - preserve project memory and continuity across sessions.
- `skill_agile_execution.md` - keep planning, execution, and validation synchronized.
- `skill_api_integration.md` - define standards for reliable external API use.
- `skill_hydrogen_research.md` - gather and structure hydrogen industry intelligence.
- `skill_langgraph_workflow.md` - define modular LangGraph workflow standards.
- `skill_project_documentation.md` - keep repository documentation clear and current.
- `skill_rag_pipeline.md` - define retrieval and historical intelligence standards.
- `skill_reliability.md` - enforce stability, observability, and recoverability.
- `skill_report_generation.md` - define executive report generation standards.
- `skill_report_generation_ops.md` - turn validated findings into concise reports.
- `source_validation_skill.md` - check freshness, credibility, and duplication.

## Usage

- Use the most specific skill that matches the task.
- Keep each skill focused on one responsibility.
- Prefer short, concrete instructions over broad prose.
- Update a skill when the workflow changes instead of adding duplicate guidance elsewhere.

## Project Notes

- Skills in this directory are treated as reusable operational guidance for the project.
- Keep them practical, current, and aligned with the active project phase.
- Preserve clarity over completeness; a skill should be easy to apply at a glance.
