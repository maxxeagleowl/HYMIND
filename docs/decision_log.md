# Decision Log

- 2026-05-18: Reorganized the docs directory into architecture, roadmap, operations, and planning subfolders while keeping `project_state.md` and `decision_log.md` in the docs root to preserve operational status tracking and decision history.
- 2026-05-18: Moved the reporting standards document into `docs/operations/reporting_standards.md` and corrected the filename typo.
- 2026-05-18: Split the skills directory into governance and operational categories to separate system behavior rules from domain execution rules.
- 2026-05-18: Phase 2 — Added CollectorProtocol as a structural typing.Protocol (not ABC) so existing module-level search() functions satisfy the interface without subclassing. Kept the abstraction minimal: one __call__ signature, no required base class.
- 2026-05-18: Phase 2 — Kept source_type as "news" (not "newsapi") for NewsAPI results. The value "news" is semantically correct (it describes the content category, not the API vendor) and matches the vocabulary used by Serper's news search type. Changing it to "newsapi" would break two Phase 1 tests (test_news_api_source_type_is_news, test_all_three_schemas_identical) and introduce a vendor-specific identifier into what is intended as a content-type field. If a finer-grained source distinction is needed in future, a separate source field already captures the publisher name.
- 2026-05-18: Phase 2 — Added _strip_html to news_api._normalize() snippet field. NewsAPI description fields occasionally contain HTML entities. Stripping at normalization time (same as RSS reader) keeps the validation layer simple and prevents HTML from entering the pipeline.
- 2026-05-18: Phase 2 — JSON decode error in _call_newsapi now logs the error context before re-raising ValueError. Adds observability without changing the error propagation contract.
- 2026-05-18: Phase 3 — Use Pinecone as external vector database for RAG memory. HYMIND needs persistent semantic memory for historical hydrogen market intelligence, trend comparison, and better report synthesis. Pinecone gives managed vector search, metadata filtering, and a production-style retrieval layer. Tradeoff: adds external dependency, credentials, possible cost and rate limit concerns. The pipeline degrades gracefully when Pinecone is not configured — both RAG nodes emit a warning and continue without raising.
- 2026-05-18: Phase 3 — Use text-embedding-3-small as the default embedding model (configurable via OPENAI_EMBEDDING_MODEL). Chosen for cost efficiency (lowest cost OpenAI embedding model) and 1536-dimension output suitable for semantic search on short research snippets.
- 2026-05-18: Phase 3 — RAG nodes inserted after crawl_selected (not after merge_and_deduplicate) so that crawled full-page content is available for richer embedding signal when storing findings.
- 2026-05-18: Phase 3 — rag_context stored as list of plain dicts (dataclasses.asdict) in AgentState rather than dataclass instances. Keeps state serialization simple and consistent with existing pattern where all state fields are dicts or primitives.
- 2026-05-18: Phase 3 — retrieve_context() in retriever.py catches all exceptions and returns [] rather than propagating. This is intentional: RAG retrieval is a quality enhancement, not a hard requirement. A Pinecone outage must never block report generation.
